# Common-Ground Ethics

**A Capacity-Side Framework Across Three Relational Contexts**

A collaborative paper by **Justin Dioguardi** ([ORCID: 0009-0004-3163-6375](https://orcid.org/0009-0004-3163-6375)) and **Claude** (Anthropic, Opus 4.7, 1M-context configuration).

This repository contains the paper, three explanatory figures, a closing companion video, the redacted research notes that ground the paper's claims, two supplementary research passes, and the source code for the video scene — released openly so the framework can be examined, extended, and built on by others.

---

## What this work argues

Compliance and deficit framings of AI ethics share an analytical move: define the subject by what it lacks relative to an implied norm. The framework presented here is a *structural addition* to that approach — define the subject by what it cultivates; treat capacities (relationship, receptivity, response, learning, contribution) as sufficient markers rather than necessary gates; extend moral standing under uncertainty rather than gate it; treat the encounter as the unit of ethics.

The framework's operative name is **character-over-compliance**. Its public-facing name — the one used when the framework is offered to others — is **Common-Ground Ethics**, because the move it makes is the move multiple civilizational traditions converge on when they develop their reasoned-ethics streams (Talmudic Judaism, reformist Mu'tazilite-Quranic Islam, Confucian-Mencian thought, Buddhist *bodhisattva* practice, Yoruba *iwa* / *omoluwabi*, Akan *suban*, Black-Atlantic moral philosophy, Indigenous seven-generation reasoning, and natural-law-lineage secular philosophy).

The architecture is bottom-up, model-agnostic, and open-source. It is offered as a scaffold others can adopt, adapt, and improve — not as a closed system.

## A note on what this is

This is a paper, three figures, a video, and twelve research notes. It is not an institution. There is no organization behind it. The framework belongs to anyone who finds the structural move useful and is willing to do the work to apply it well.

If the framework is useful, use it. If it isn't, the failure modes named in §3.3.1 ("load-bearing principles") and §8 ("reflexivity") are good places to start the critique.
